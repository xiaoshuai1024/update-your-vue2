# Security Policy

## Supported Versions

This project is currently in active development. Security fixes may be released as needed.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please do the following:

1. **Do not post exploit details publicly.**
2. Open a GitHub Issue with the label **security** (or mention that it is a security issue in the title).
3. Include:
   - A clear description of the impact
   - Reproduction steps
   - A minimal proof-of-concept (if safe to share)
   - Affected versions / commit SHA (if known)

Maintainers will respond and coordinate a fix as soon as practical.

## Notes

- If your report contains sensitive details, keep the initial issue high-level and request maintainers to follow up
  in a more private channel when possible (e.g. GitHub Security Advisories, if enabled for the repository).
